# 评分标准： 总分 100分

评分维度以及标准：                             
1. 读取文件数据
优秀（11-15）： 基本满足所有的要求，读取文件没有错误
良好（6-10）： 读取文件有一些问题比如重复读取等等
一般（0-5）： 存在较大的问题，或者代码本身有重要的问题

2. 可视化分析 
优秀（11-15）： 图清晰可理解，代码没有什么问题
良好（6-10）： 画出的图不是特别容易理解意思
一般（0-5）： 图的不正确，代码有误

3. 文本预处理
优秀（21-25）： 按照步骤完成了所有的过程
良好（11-20）： 其中的一两步有问题，但整体来讲可以运行
一般（0-10）： 代码缺失，预处理严重不对

4. 把文本转换成向量     
优秀（11-15）： 正确调用了tf-idf向量的转换，没有大问题
良好（6-10）： 在测试数据上也做了fit_transform，或者其他类似的错误
一般（0-5）： 有比较严重的问题

5. 利用模型去训练               
优秀（11-15）： 三个模型用的都正确，而且GridSearchCV的用法也正确
良好（6-10）： 其中的一个模型用得不对，或者交叉验证用法不太对
一般（0-5）： 代码存在严重的问题

6. 思考题              
优秀（4-5）： 回答得合理，基本上符合正确的逻辑
良好（2-3）： 给出了一些点子，但可能还离答案有点距离
一般（0-1）： 没有答对

7. 代码质量，是否按标准完成         
优秀（7-10）： 代码清晰、可读性强、而且效率高
良好（4-6）： 注释不完整、可读性较弱、代码有些地方效率低
一般（0-3）： 可读性弱、无注解、文件的结构不清晰
